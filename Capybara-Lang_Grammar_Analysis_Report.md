# Capybara-Lang 문법 분석 보고서

## 1. 개요

**파일 확장자**: `.bara`  
**분석 일자**: 2025년 9월 25일

## 2. 문법 구조 분석

### 2.1 주석 시스템

#### 단일 행 주석
```
`이것은 주석 입니다.`
```
- **구문**: 백틱(\`) 사이에 텍스트를 감싸서 주석 생성
- **특징**: 한국어 주석을 네이티브로 지원

#### 멀티라인 주석
```
```
이것은 줄넘김이 있는 주석 입니다.
줄
넘
김
```
```
- **구문**: 삼중 백틱(\`\`\`) 블록으로 여러 줄 주석
- **특징**: 마크다운과 유사한 블록 주석 구조

### 2.2 변수 선언 및 할당

#### 기본 변수 선언 (set 키워드)
```
set;["변수이름"]:("할당할값")
```

**예시들:**
- `set;["ex1"]:("안녕 문자열1")` - 문자열 할당
- `set;["ex2"]:(100)` - 숫자 할당
- `set;["ex6"]:({"json": "good"})` - JSON 객체 할당

#### 타입 지정 변수 선언 (var 키워드)
```
var;["변수이름"]:<타입>;<("값")>
```

**예시:**
- `var;["exex"]:<str>;<("wow! ex!")>` - 문자열 타입 명시

### 2.3 데이터 타입

#### 불린(Boolean) 타입
```
set;["ex3"]:(True)
set;["ex4"]:(true)  
set;["ex5"]:(TrUe)
```
- **특징**: 대소문자 구분 없이 true/false 인식
- **값**: `true`, `True`, `TrUe`, `false` 등 모든 형태 지원

#### 문자열(String) 타입
- **구문**: `("문자열 내용")`
- **예시**: `("안녕 문자열1")`, `("값을 입력하세요: ")`

#### 숫자(Number) 타입
- **구문**: `(숫자)`
- **예시**: `(100)`

#### JSON 객체 타입
- **구문**: `({"key": "value"})`
- **예시**: `({"json": "good"})`

### 2.4 입출력 시스템

#### 입력 받기
```
<변수이름>;<input;print:("프롬프트 메시지")>
```

#### 직접 출력
```
<print;input;print:("프롬프트 메시지")>
```

#### 단순 출력
```
<print:("출력할 내용")>
```

### 2.5 조건문 구조

#### if문
```
if {조건식}
<-
<실행문>
->
```

#### elif문 (fi 키워드)
```
fi {조건식}
<-
<실행문>
->
```

#### else문 (el 키워드)
```
el {}
<-
<실행문>
->
```

**완전한 조건문 예시:**
```
if {<pop> = true}
<-
<print:("참")>
->
fi {<pop> = false}
<-
<print:("거짓")>
->
el {}
<-
<print:("참 이나 거짓만 입력")>
->
```

## 3. 문법 규칙 정의

### 3.1 구문 분석 규칙

1. **세미콜론(;) 구분자**: 대부분의 구문 요소는 세미콜론으로 구분
2. **대괄호 []: 변수명 식별자**
3. **소괄호 (): 값 또는 표현식 래핑**
4. **각괄호 <>: 함수 호출 및 연산자**
5. **중괄호 {}: 조건식 및 블록 구분**
6. **화살표 <-, ->: 블록 시작/끝 표시**

### 3.2 키워드 분류

#### 변수 관련
- `set`: 기본 변수 선언
- `var`: 타입 지정 변수 선언

#### 조건문 관련
- `if`: 조건문 시작
- `fi`: elif 구문 (else if)
- `el`: else 구문

#### 함수/연산자 관련
- `print`: 출력 함수
- `input`: 입력 함수
- `pop`: 값 추출/접근 연산자

### 3.3 타입 시스템

#### 지원하는 타입
- `<str>`: 문자열 타입
- 불린: `true`/`false` (대소문자 무관)
- 숫자: 정수 지원 확인
- JSON: 객체 리터럴 지원

## 4. 특징 및 독창성

### 4.1 언어의 독특한 특징

1. **한국어 친화적**: 주석에서 한국어를 자연스럽게 지원
2. **유연한 대소문자**: 불린 값에서 대소문자 구분 없음
3. **독특한 구문 구조**: 세미콜론과 다양한 괄호를 조합한 고유한 문법
4. **블록 구분자**: `<-`와 `->` 화살표를 사용한 직관적인 블록 표현

### 4.2 문법의 장점

1. **가독성**: 화살표 블록 구분자로 코드 블록이 명확함
2. **타입 안전성**: 명시적 타입 지정 가능
3. **표현력**: JSON 리터럴 직접 지원
4. **직관적 조건문**: `fi`(elif), `el`(else) 등 축약된 키워드

### 4.3 개발 상태

- **구현 언어**: Rust (Cargo.toml 기반)
- **버전**: 0.1.0 (초기 개발 단계)
- **에디션**: 2024 (최신 Rust 에디션 사용)

## 5. 문법 요약 (BNF-스타일)

```
program ::= statement*

statement ::= comment | variable_declaration | conditional | expression

comment ::= '`' text '`' | '```' text '```'

variable_declaration ::= 
    'set' ';' '[' identifier ']' ':' '(' value ')' |
    'var' ';' '[' identifier ']' ':' '<' type '>' ';' '<' '(' value ')' '>'

conditional ::= if_block elif_block* else_block?

if_block ::= 'if' '{' condition '}' '<-' statement* '->'
elif_block ::= 'fi' '{' condition '}' '<-' statement* '->'
else_block ::= 'el' '{}' '<-' statement* '->'

expression ::= '<' function_call '>'

function_call ::= 'print' ':' '(' value ')' | 'input' ';' 'print' ':' '(' value ')'

value ::= string | number | boolean | json_object

type ::= 'str' | ...
```
